CCswarm4Manus

基于 Thin Agent / Fat Platform 架构的多智能体 Claude Code 编排平台

CCswarm4Manus 是一个高效的多智能体协作编排系统，通过 Master Planner 层统一调度远程 Claude Code 节点， 实现任务的并行下发、状态探针监控与质量门控。系统采用 SSH + tmux 守护进程实现零阻塞异步调度， 让复杂任务被拆解后并行执行，大幅提升 Claude Code 的任务处理能力。

架构设计

四层核心架构

分层设计，各司其职，构建可扩展的多智能体编排系统

Layer 1

Master Planner Layer (Manus)

负责意图解析、任务拆解、宏观状态管理全局任务调度与监控 polling

Layer 2

Remote Communication Layer

SSH + tmux 守护进程实现零阻塞异步下发，ControlMaster 复用连接

Layer 3

Node Execution Layer

Claude Code CLI 无头模式运行，角色隔离的 Thin Agent，工具权限精确管控

Layer 4

Quality Gate Layer

Hooks 生命周期钩子拦截保存/退出操作，强制运行 Lint + 单元测试

核心特性

为什么选择 CCswarm4Manus

精心设计的特性，确保多智能体协作高效可靠

零阻塞异步调度

tmux 守护进程实现任务下发后立即释放上下文，无需等待远程执行完成即可处理下一个任务

轻量级状态探针

响应 JSON 小于 100 bytes，内置死循环检测机制，快速感知节点状态变化

并发安全状态管理

flock 文件锁机制保护 claude-progress.json，支持多进程并发读写不冲突

优雅中断机制

SIGINT 优先发送，15秒 Checkpoint 保存窗口，自动保存进度后退出

基于角色的权限隔离

Planner/Coder/Reviewer 三种角色，工具互斥矩阵精确管控，防止越权操作

src/state_manager.js

并发安全的 JSON 状态读写核心模块，提供 initializeState()、updateTaskStatus()、readState() 接口，使用 flock 文件锁防止并发写入冲突

scripts/spawn_agent.sh

异步任务下发脚本，通过 tmux 守护进程启动远程 Claude Code CLI，支持锁文件机制防止重复下发，任务ID追踪

scripts/probe_status.sh

轻量状态探针脚本，返回极简 JSON 状态信息（小于100 bytes），内置死循环检测，超时自动标记

scripts/interrupt_agent.sh

优雅中断脚本：优先发送 SIGINT 信号，保留15秒 Checkpoint 保存窗口，自动更新状态为 interrupted

scripts/set_dirty_bit.sh

PostToolUse Hook 脚本，当检测到 Edit 或 Write 工具调用时设置 dirty bit，标记代码已被修改

scripts/verify_before_exit.sh

Stop Hook 退出前质量门控脚本，强制运行 lint/单元测试，非零退出码阻止 Agent 退出并汇报失败

.claude/settings.json

Claude Code Hooks 配置中心，定义 PostToolUse、Stop 等生命周期钩子绑定，角色权限矩阵
